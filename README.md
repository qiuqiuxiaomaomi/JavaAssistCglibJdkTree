代理：
        代理是一种常用的设计模式，其目的就是为其它对象提供一个代理以控制对某个对象的访问。
     代理类负责为委托类预处理消息，过滤消息并转发消息。

        为了保持行为的一致性，代理类和委托类通常会实现相同的接口。通过代理类这中间这一层，能
     有效控制对委托类对象的直接访问，可以很好的隐藏和保护委托类对象，同时也为实施不同控制
     策略预留了空间，从而在设计上获得更大的灵活性。
